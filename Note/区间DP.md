# 区间DP 学习笔记

## 关于转移时费用的计算

存在两种情况：

### 价值的计算符合结合律

对于这种情况，我们可以建立一个数组来代表某个区间内的价值，转移时直接使用这个数组来计算费用。

### 价值的计算不符合结合律

对于这种情况，我们可以在 dp 数组中存储一些额外的信息（例如存储区间在某种操作下的价值），以便在转移时能够正确计算费用。同时，在枚举时应当同时枚举这个额外的信息，以确保转移的正确性。

实战技巧：

* 如果看到区间求和、区间 GCD、区间最大值、位运算作为合并代价，直接冲 $O(n^3)$ 预处理 $v[i][j]$ 。
* 如果看到代价与相邻元素的状态（如颜色、字符是否匹配）有关，先检查是否需要增加 $dp[i][j][state]$ 状态 。
* 如果看到回文性质，通常不需要显式的 $cost$ 数组，而是通过 $s[i] == s[j]$ 的布尔判断来触发不同的转移逻辑 。
