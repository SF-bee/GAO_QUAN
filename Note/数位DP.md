# 📝 数位 DP 核心学习笔记

## 1. 适用场景

1. 要求统计满足一定条件的数的数量（即，最终目的为计数）。

2. 这些条件经过转化后可以使用「数位」（就是说限制于数位相关）的思想去理解和判断。

3. 输入会提供一个数字区间（有时也只提供上界）来作为统计的限制。

4. 上界很大（比如 $10^{18}$），暴力枚举验证会超时。

---

## 2. 核心思想

考虑人数数的方法：最朴素的计数就是从小到大开始依次加一．但我们发现对于位数比较多的数，这样的过程中有许多重复的部分．例如，从 7000 数到 7999、从 8000 数到 8999、和从 9000 数到 9999 的过程非常相似，它们都是后三位从 000 变到 999，不一样的地方只有千位这一位，所以我们可以把这些过程归并起来，将这些过程中产生的计数答案也都存在一个通用的数组里。换句话说，就是对一个位置的数每一位填充，然后用一个数组代表填到每一位的答案。此数组根据题目具体要求设置状态，用递推或 DP 的方式进行状态转移。

---

## 解题套路

```cpp
int dfs(int pos, int state, bool limit, bool lead)
```

这个搜索就是一切数位 dp 的板子，基本上所有的数位 dp 都是用这个改的。

1. *位置参数* 描述当前是从高到低的第几位，即阶段
2. *限制参数* 例如：题目要求相邻位不能相等，那么就要记录上一位的填了什么数。
3. *是否上界* 用于记录上一个数是不是填到上界了，以判断我当前位可以填什么。
4. *是否前导零* 在某些时候，我们需要满足一个与前一个数相关的限制，很显然，前导零不应该参与计数。

对于求解某个区间内的答案，我们一般会用前缀和原理统计。
