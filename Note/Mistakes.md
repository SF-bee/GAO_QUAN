## [L-ERR] 图论-最小环逻辑漏洞（2026-06-13）

**题目**: 学校景点最优路线（求无向图最小环）

**错误症状**: WA

**根本原因**: Floyd-Warshall 后直接枚举边 $(i,j,w)$ 计算 `w + g[i][k] + g[k][j]`，但 `g[i][k]` 和 `g[k][j]` 的最短路可能本身包含边 $(i,j)$，导致该边被计算了多次。

**反例**:
```
边 1-2: 100
边 1-3: 1
边 3-2: 1
```
Floyd 后 `g[1][2] = 2`，但对边 (1,3) 计算时：
- `f[1][3] + g[1][2] + g[2][3] = 1 + 2 + 1 = 4`（错）
- 实际边 (1,3) 在这个"路径"中被走了 2 次

**修复方案**: 对每条直接边单独隔离处理，计算不经过该边的 u→v 最短路，或使用三层 Floyd 的分层思想。
